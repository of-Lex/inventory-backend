# Серверная часть системы предметов и инвентаря
У игроков есть личные характеристики, инвентарь и наложенные эффекты. Игроки могут выбрасывать предметы из инвентаря на землю, поднимать с земли (в определённом радиусе) и использовать их. Поднятие, использование и выбрасывание разных предметов имеет свою логику (зависит от конкретного предмета). В мире есть лежащие на земле предметы, которые, в дополнение к обычным предметам, имеют позицию и время жизни. Эффекты могут создаваться обособленно, но применяться могут для нескольких игроков сразу, имеют время жизни и свою логику (зависит от конкретного эффекта). Покрыто автоматическими интеграционными тестами.

## Использование

### Требования
Для установки и запуска проекта, необходим [Node.js](https://nodejs.org/) версии 14.x или выше.

### Клонирование репозитория
Для клонирования репозитория, выполните команду:
```sh
$ git clone https://github.com/of-Lex/inventory-backend.git
```

### Перереход в папку проекта
Для перехода в папку проекта, выполните команду:
```sh
$ cd inventory-backend
```

### Установка зависимостей
Для установки зависимостей, выполните команду:
```sh
$ npm install
```

### Создание билда
Чтобы выполнить сборку, выполните команду:
```sh
npm run build
```

### Запуск
Чтобы запустить, выполните команду:
```sh
npm start
```

## Обзор архитектуры

### storage
- Storage - singltone класс, имитирует взаимодействия с базами данных

### config
- Config - singltone класс, распространяет на отдельные сущности/системы методы: get для получения, registerDefault для регистрации кастомных конфигов, хранит настройки в файле config.json с возможностью его редактирования в будущем

### core
- GameLoop - singltone класс, создаёт единственный Game Loop для всей "игры" с возможностью подписки отдельных систем
- World - создаёт мир, для корректной работы должен быть подписан на GameLoop
- Utils - статический класс со вспомогательными функциями
- Tickable - интерфейс, описывающий распространяемый tick
- Player - игрок, имеет id, имя, здоровье, броню, кол-во патронов, позицию, инвентарь, наложенные эффекты
- Item - предмет, имеет id, amount (в данный момент используется для хранения в инвентаре игрока)
- GroundItem - предмет "на земле", наследуется от Item, дополнительно имеет позицию в игровом мире и оставшееся время жизни
- Effect - эффект игрока, имеет оставшееся время жизни

### itemTypes
- ItemType - интерфейс, описывающий тип предмета
- Registrator - регистрирует предметы из папки types (не автоматически, подробнее: [Возможные улучшения](#возможные-улучшения))
- types - папка, содержащая конкретные типы предметов

### effectTypes
- EffectType - интерфейс, описывающий тип эффекта
- Registrator - регистрирует эффекты из папки types (не автоматически, подробнее: [Возможные улучшения](#возможные-улучшения))
- types - папка, содержащая конкретные типы эффектов

### types
- Position - интерфейс описывающий позицию x, y (подобие Vector2)

## Возможные улучшения
- [ ] Вместо return; и return false; во многих аспектах API лучше отдавать конкретный кастомный Error с текстом ошибки и логированием
- [ ] Поиски линейной сложности O(N) на реальном продукте можно оптимизировать по draw distance и/или чанкам игровой карты, но это должен делать отдельный игровой Streamer
- [ ] Registrator предметов/эффектов можно немного автоматизировать по папке types, чтобы достаточно было добавить только новый тип предмета
- [ ] medkit при использовании восстанавливает player.health до настраиваемого максимума. Можно дополнительно ввести config "medkitHealAmount", чтобы аптечка восстанавливала конкретное количество здоровья
- [ ] armor поднимает player.armor до неопределённого значения (вероятно, до 100). Лучше будет хранить степень износа бронежилета в самом предмете, а не в статистике игрока
- [ ] flare по хорошему должен подсвечивать в том числе и тех игроков, которые зашли в его радиус действия уже после активации, а не только тех игроков, которые были рядом с ним в момент активации, как в текущей реализации
